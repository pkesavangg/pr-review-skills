Push the current branch and create a pull request using the GitHub CLI.

This skill invokes `/gen-pr-description-template` to consolidate template logic and eliminate duplication.

Inputs available: ISSUE_ID, ISSUE_SUMMARY, list of changed files, base branch chosen by user, coverage results (if task involves unit/UI tests)

## Instructions

1. Ask the user:
   > "Which branch should I raise the PR against? (default: `main`)"

2. Gather PR description context:
   - Extract ISSUE_ID from current branch name
   - Run `git log main..HEAD --oneline` to get commit messages
   - Run `git diff main...HEAD --stat` to get changed files
   - Identify change type (Feature, Refactor, Bug fix, Test, Dependency)

3. Invoke `/gen-pr-description-template` with the context:
   - ISSUE_ID
   - Change type
   - Changed files list
   - Commit messages
   - Test changes info (if any)

4. Push the branch:
   ```
   git push -u origin {branch-name}
   ```

5. Create the PR using `gh pr create`:
   ```bash
   gh pr create \
     --title "{ISSUE-ID} {issue summary}" \
     --body "{PR description from gen-pr-description-template}" \
     --base {base-branch}
   ```

   - `title`: Use format `{ISSUE-ID} {issue summary}` (e.g., `MA-3316 Enable Bluetooth pairing for scales`)
   - `body`: Use the description generated by `/gen-pr-description-template`
   - `base`: Branch specified by user (default: `main`)

6. Return the PR URL to the user.

7. Transition the Jira issue to **In Review**:
   - Call `getAccessibleAtlassianResources` to retrieve the `cloudId` (if not already known)
   - Call `getTransitionsForJiraIssue` with `cloudId` and `ISSUE_ID` to list available transitions
   - Find the transition whose name contains "In Review" (case-insensitive)
   - Call `transitionJiraIssue` with that transition's `id`
   - Confirm the status was updated

## Rules

- ✅ Use `/gen-pr-description-template` to generate PR body (single source of truth)
- ✅ Push with `-u` flag to track remote branch
- ✅ Use `gh pr create` with proper title format
- ✅ Transition Jira issue to "In Review" after PR is created
- ❌ Do NOT include `Co-Authored-By` line in PR body (template handles this)
- ❌ Do NOT manually build PR description inline (let `/gen-pr-description-template` do it)
