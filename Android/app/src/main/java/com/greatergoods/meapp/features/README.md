# Features Module

Contains feature-specific code organized by feature modules.

## Structure

- **common/**: Shared UI components and logic across features
- **auth/**: Authentication flows (Login, Register, etc.)
- **entry/**: Onboarding and splash screens
- **settings/**: User preferences and settings

Each feature follows the structure:

- model/
- enum/
- strings/
- viewmodel/
- view/
  - screens/
  - components/

# Common Feature Module

This directory is intended for shared sample screens and utilities that demonstrate or provide common functionality across the app, such as theme token usage and dynamic theming samples.
